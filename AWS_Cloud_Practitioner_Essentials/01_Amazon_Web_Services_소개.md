# 클라우드 컴퓨팅 배포 모델

## 클라이언트 및 서버 모델
- 고객: 클라이언트. 클라이언트는 사람이 컴퓨터 서버에 요청을 하기 위해 상호 작용하는 웹 브라우저 또는 데스크톱 애플리케이션
- 서버: EC2같은 가상 서비스.
=> 서버는 고객의 요청을 평가하고 클라이언트에 정보를 반환하여 요청 이행

## 클라우드 컴퓨팅이란
- 인터넷을 통해 IT 리소스와 애플리케이션을 온디맨드로 제공하는 서비스

### 클라우드 컴퓨팅 배포 모델
- 원하는 클라우드 애플리케이션 구성 요소, 선호 리소스 도구, 레거시 IT 인프라 요구 사항 등의 요소를 고려해야 함
#### 클라우드 기반
  - 기존 애플리케이션을 클라우드로 마이그레이션 하거나 클라우드에서 새 애플리케이션을 설계하고 구축할 수 있음. 
  - 애플리케이션은 IT 팀의 관리가 필요한 하위 수준 인프라에 구축할 수도 있고, 핵심 인프라의 관리, 설계, 크기 조정 요구 사항을 줄여주는 상위 수준의 서비스를 사용하여 구축할 수도 있음
#### 온프레미스
  - 프라이빗 클라우드 배포라고도 함
#### 하이브리드
  - 클라우드 기반 리소스가 온프레미스 인프라에 연결됨

# 클라우드 컴퓨팅의 이점
`가변 비용`: 초기 비용을 가변 비용으로 바꿀 수 있음
`비용 최적화`: 데이터 센터의 운영 및 유지 관리에 비용을 지출할 필요가 없음
  -> 애플리케이션과 고객에 더 집중 가능
`용량`
-> 필요한 인프라 용량을 추정할 필요가 없으며, 수요에 따라 확장 또는 축소할 수 있음
`규모의 경제`
-> 수많은 고객 사용량이 누적되므로 규모의 경제 달성 가능
`속도 및 민첩성`
`몇 분만에 전세계에 배포`
