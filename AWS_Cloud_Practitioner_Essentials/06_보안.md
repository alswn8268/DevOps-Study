## 공동 책임 모델
> 사용자는 인스턴스 운영 체제, 호스트 기반 방화벽, 애플리케이션, 계정 관리, 보안 그룹에 대해, AWS는 데이터 센터의 물리적 보안, 네트워크 인프라, 하드웨어 및 소프트웨어 인프라, 가상화 인프라에 대해 보안을 담당

-> AWS, 사용자 모두 보안에 대한 책임을 짐
## AWS Identity and Access Management (IAM)
- 사용자 자격 증명을 생성하고 권한을 할당하기 위해 사용
- 회사의 고유한 운영 및 보안 요구 사항에 따라 액세스 권한을 구성할 수 있는 유연성 제공
  - IAM 사용자, 그룹 및 역할
  - IAM 정책
  - 다중 인증
### AWS 계정 루트 사용자
> AWS 계정을 처음 만들면 루트 사용자 자격 증명이 설정됨. 이후 IAM 사용자를 생성하고 루트 사용자는 사용하지 않음
### IAM 사용자
- 사용자가 AWS에서 생성하는 자격 증명

### IAM 그룹
- IAM 사용자 그룹

### IAM 정책
- AWS 서비스 및 리소스에 대한 권한을 허용하거나 거부하는 문서
- **최소 권한** 보안 원칙을 따라야 함
- JSON 언어로 작성이 됨

### IAM 역할
- 임시로 권한에 액세스하기 위해 수임할 수 있는 자격 증명

### 다중 인증 (MFA)
- AWS 계정에 추가 보호 계층을 제공

## AWS Organizations
- 고객이 중앙 위치에서 여러 AWS 계정을 통합하고 관리할 수 있도록 지원
- 서비스 제어 정책(SCP)를 사용하여 조직의 계정에 대한 권한을 중앙에서 제어함
ex) 조직 단위 (OU) 그룹화

## 규정 준수
### AWS Artifact
- AWS 보안 및 규정 준수 보고서와 일부 온라인 계약에 온디맨드로 액세스할 수 있는 서비스
- Artifact 계약 / Artifact 보고서 이렇게 두 섹션이 있음

#### 고객 규정 준수 센터
- AWS 규정 준수에 대해 자세히 알아볼 수 있는 리소스들이 포함됨

## 애플리케이션 보안
### AWS WAF (웹 방화벽)
- 일반적인 웹 취약점 공격에서 웹 애플리케이션이나 API를 보호하는데 도움이 되는 웹 애플리케이션 방화벽
- SQL Injection, XSS, 특정 트래픽 패턴 등 필터링
#### 규칙
- 규칙을 사용하여 일치하는 요청을 차단하거나 허용할 수 있음

### AWS Sheild
- DDoS 공격으로부터 애플리케이션을 보호하는 서비스
- Standard / Advanced

### Amazon Inspector
- 애플리케이션에 대해 자동화된 보안 평가를 수행

## 추가 보안 서비스
### Key Management Service (KMS)
- 고객이 사용하면 암호화 키를 사용하여 암호화 작업을 수행하도록 지원
- 키에 필요한 특정 수준의 액세스 제어를 선택할 수 있음
### Amazon GuardDuty
- AWS 제품 및 서비스에 대한 지능형 위협 탐지 기능 제공
