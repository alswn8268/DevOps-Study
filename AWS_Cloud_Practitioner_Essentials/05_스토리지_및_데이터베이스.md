## AWS 스토리지
### 블록 스토리지
- 동일한 크기의 데이터 조각(블록)으로 구분
- 블록 스토리지는 EC2 인스턴스에서 실행되는 애플리케이션에 사용됨
- 블록 수준 스토리지 볼륨은 물리적 하드 드라이브처럼 동작하므로, 파일이 수정되면 변경된 부분만 업데이트됨
#### 인스턴스 스토어
- EC2 인스턴스에 임시 블록 수준 스토리지 제공
- 가상 서버이므로, 인스턴스를 중지된 상태에서 시작하면 인스턴스가 다른 호스트에서 시작할 수 있음 => 장기적으로 필요하지 않은 임시 데이터가 포함된 사용 사례에 사용 
#### Amazon EBS 볼륨
- EC2 인스턴스에서 사용할 수 있는 블록 수준 스토리지 볼륨을 제공하는 서비스
- 인스턴스를 중지 또는 종료하더라도 연결된 EBS 볼륨의 모든 데이터를 사용할 수 있음
> EBS 스냅샷은 증분 백업으로, 처음 볼륨을 백업하면 모든 데이터가 복사되고 이후 백업에서는 가장 최근의 스냅샷 이후 변경된 데이터 블록만 저장됨

### 객체 스토리지
- 각 객체는 데이터, 메타데이터, 키로 구성됨
#### Amazon Simple Storage Service (Amazon S3)
- 객체 수준 스토리지를 제공하는 서비스. 데이터를 버킷 내에 객체 단위로 저장
- 사용한 만큼만 비용 지불
#### 객체 스토리지 클래스
> Standard, Standard-IA, One Zone-IA, Intelligent-Tiering, Glacier Instant Retrieval, Glacier Flexible Retrieval, Glacier Deep Archive

### 파일 스토리지
- 여러 클라이언트가 공유 파일 폴더에 저장된 데이터에 엑세스할 수 있음
#### Amazon Elastic File System (Amazon EFS)
- AWS Cloud Services 및 온프레미스 리소스에 사용되는 확장 가능한 파일 시스템
- 저절로 규모가 확정되거나 축소됨

> EBS와 EFS의 차이: EBS 볼륨은 가용 영역 수준 리소스라서 단일 가용 영역에 데이터를 저장함. EC2 인스턴스를 EBS 볼륨에 연결하려면 EC2 인스턴스와 EBS 볼륨 모두 동일 가용 영역에 상주해야 함. 한편, EFS는 기본적으로 여러 가용 영역 내 여러 가용 영역 간에 데이터를 저장하여, 파일 시스템이 위치한 리전의 모든 가용 영역에서 동시에 데이터를 액세스 할 수 있음

## AWS 데이터베이스
### 관계형 데이터베이스(RDB)
: 데이터가 다른 데이터 부분과 관련된 방식으로 저장됨. SQL 사용
#### Amazon Relational Database Service (Amazon RDS)
- AWS 클라우드에서 관계형 데이터베이스를 실행할 수 있는 서비스
- 하드웨어 프로비저닝, DB 설정, 패치 적용 등과 같은 작업 자동화
- 다양한 보안 옵션 제공
ex) Amazon Aurora, PostgreSQL, MySQL 등 7개

#### Amazon Aurora (Amazon Aurora)
- 엔터프라이즈급 관계형 데이터베이스
- MySQL과 PostgreSQL 등과 호환되며 최대 3~5배 빠름.
- 불필요한 IO 작업을 줄여 DB 비용 절감
- 3개의 가용 영역에 6개의 데이터 복사본을 복제

### 비관계형 데이터베이스 (NoSQL)
- 키-값 쌍 구조로 구성된 데이터베이스
#### Amazon DynamoDB
- 키-값 데이터베이스 서비스로, 어떤 규모에서도 10ms 미만의 성능 제공
- 서버리스이므로 서버를 프로비저닝, 패치 적용 및 관리 필요 X
- 데이터베이스 크기가 축소 또는 확장되면 DynamoDB는 용량 변화에 맞춰 자동으로 크기를 조정하면서도 일관된 성능 유지

### AWS Database Migration Service (AWS DMS)
- 관계, 비관계형 데이터베이스 및 기타 유형의 데이터 스토어를 마이그레이션할 수 있음
- 원본 DB와 대상 DB는 유형이 동일할 필요 없고, 마이그레이션하는 동안 원본 DB도 계속 작동하므로 DB를 사용하는 애플리케이션의 가동 중지 시간을 줄일 수 있음


## 추가 데이터베이스 서비스
`Amazon Redshift`: 데이터 웨어하우스에서 빅데이터를 쿼리 및 분석
`Amazon DocumentDB`: MongoDB 워크로드를 지원하는 문서 DB 서비스
`Amazon Neptune`: 고도로 연결된 데이터 세트를 사용하는 애플리케이션을 빌드하고 사용하는 그래프 데이터베이스 서비스
`Amazon Quantum Ledger Database (Amazon QLDB)`: 애플리케이션 데이터에 발생한 모든 변경 사항의 전체 기록을 검토하는 원장 DB
`Amazon Managed Blockchain`: 오픈 소스 프레임워크를 사용하여 블록체인 네트워크를 생성하고 관리하는데 사용할 수 있는 서비스. 여러 당사자가 중앙기관 없이 거래를 실행하고 데이터를 공유할 수 있는 분산형 원장 시스템
`Amazon ElastiCache`: 자주 사용되는 요청의 읽기 시간을 향상시키기 위해 DB 위에 캐싱 계층을 추가하는 서비스로 Redis, Memcached를 지원함
`Amazon DynamoDB Accelerator (DAX)`: DynamoDB용 인메모리 캐시
