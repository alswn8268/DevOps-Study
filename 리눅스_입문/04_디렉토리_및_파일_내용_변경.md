# 4.1 생성, 이동, 복사
## 복사
```
cp option source(s) target
```
### 파일 복사
- `cp` 명령은 파일의 내용을 다른 파일로 복사하는 명령어
- 기본적으로 덮어쓰며, `-i` 옵션을 통해 덮어쓰기 방지 가능

### 디렉터리 복사
- `cp` 명령어를 사용하여 디렉터리를 다른 디렉터리로 복사 가능. **반드시 `-r` 옵션을 사용해야 함**

|옵션|설명|
|---|---|
|-i| 대화형 모드, 복사시 파일을 덮어쓰는 것을 방지|
|-r|디렉터리 복사시, 해당 디렉터리의 하위 모든 파일과 디렉터리를 새로운 디렉터리에 복사|

## 이동, 이름 변경
- `mv` 명령어로 파일을 다른 디렉터리로 이동
```
mv option source target
```
## 생성
- 파일을 생성할 때는 `touch` 명령어를, 디렉터리를 생성할 때는 `mkdir` 명령어를 사용함
### 빈 파일 생성
```
touch filename
```
### 디렉터리 생성
- 한 번에 계층적으로 생성하려면 `-p` 옵션을 사용해야 함
- 아규먼트를 여러 개 지정하면 여러 디렉터리를 동시에 생성할 수 있음
```
mkdir dirname
```
## 삭제
- 파일을 삭제할 때는 `rm` 명령어를, 디렉터리를 삭제할 때는 `rmdir` 명령어를 사용함
- `-i` 옵션을 추가하면 진짜 삭제할 것인지 여부를 물어봄
```
rm option filename
```
### 디렉터리 삭제
1) 빈 디렉터리인 경우
```
rmdir directory(s)
```
2) 파일이 있는 디렉터리인 경우
```
rm -r directory(s)
```

# 4.2 링크 
- 특정 하나의 파일 또는 디렉터리에 접근을 쉽게할 수 있는 방법
## 링크 기본
### 하드 링크
  - 특정 파일이 디스크에 저장되어 있는 위치가 동일한 곳을 가리키고 있는 포인터
  - 실제 디스크의 위치를 똑같이 가리키고 있기 때문에 용량이 더 늘어나지 않음
  - 디렉터리에 대해서는 하드 링크를 생성할 수 없음
  - `ls -l` 명령어로 확인
  ```
  [vagrant@user03 ~]$ ls -il /bin/cd
  17931 -rwxr-xr-x. 1 root root 30 Apr 30  2024 /bin/cd
  [vagrant@user03 ~]$ ls -il /usr/bin/cd
  17931 -rwxr-xr-x. 1 root root 30 Apr 30  2024 /usr/bin/cd
  ```
  => 두 번째 값인 inode가 같다는 것은 같은 디스크의 같은 위치를 가리키고 있다는 것을 의미

### 심볼릭 링크
  - 특정 파일 또는 디렉터리를 가리키고 있는 포인터 (=바로가기)
  - `ls -F` 명령어에서 파일 이름 마지막에 `@` 문자가 있는 파일이며, `ls -l` 명령어에서는 파일의 형식 필드에 l이라고 표시됨
  ```
  [vagrant@user03 ~]$ ls -F /bin
  /bin@
  [vagrant@user03 ~]$ ls -l /bin
  lrwxrwxrwx. 1 root root 7 Nov  3  2024 /bin -> usr/bin
  ```
  => `ls -l` 명령어 결과애서 심볼릭 링크의 파일 이름을 확인해 보면 화살표(->) 다음은 실제 포인터로 가리키고 있는 파일의 이름을 나타냄

  ## 링크 생성 및 삭제
  ### 하드 링크 생성 및 삭제
  하드 링크를 생성하기 위해서는 `ln` 명령어를, 삭제하기 위해서는 `rm` 명령어를 사용
  ```
  ln source target
  ```
  - 하드 링크 생성
  ```
  [vagrant@user03 ~]$ ls -il fileA
  1208351 -rw-r--r--. 1 vagrant vagrant 0 Oct 17 08:13 fileA
  [vagrant@user03 ~]$ ln fileA hardA
  [vagrant@user03 ~]$ ls -il fileA hardA
  1208351 -rw-r--r--. 2 vagrant vagrant 0 Oct 17 08:13 fileA
  1208351 -rw-r--r--. 2 vagrant vagrant 0 Oct 17 08:13 hardA
  ```
  - 하드 링크 삭제
  ```
  [vagrant@user03 ~]$ ls -il fileA
  1208351 -rw-r--r--. 1 vagrant vagrant 0 Oct 17 08:13 fileA
  [vagrant@user03 ~]$ ls -il hardA
  ls: cannot access 'hardA': No such file or directory
  ```

  ### 심볼릭 링크 생성 및 삭제
  심볼릭 링크를 생성하기 위해서는 `ln -s`를, 삭제하기 위해서는 `rm` 명령어를 사용

  - 심볼릭 링크 생성
  ```
  [vagrant@user03 ~]$ ls -il fileA
  1208351 -rw-r--r--. 1 vagrant vagrant 0 Oct 17 08:13 fileA
  [vagrant@user03 ~]$ ln -s fileA symbolA
  [vagrant@user03 ~]$ ls -il fileA symbolA
  1208351 -rw-r--r--. 1 vagrant vagrant 0 Oct 17 08:13 fileA
  1208372 lrwxrwxrwx. 1 vagrant vagrant 5 Oct 17 08:16 symbolA -> fileA
  ```
  - 심볼릭 링크 삭제
  ```
  [vagrant@user03 ~]$ rm symbolA
  [vagrant@user03 ~]$ ls -il fileA symbolA
  ls: cannot access 'symbolA': No such file or directory
  1208351 -rw-r--r--. 1 vagrant vagrant 0 Oct 17 08:13 fileA
  ```
