# 2.1 유닉스 / 리눅스 시스템 환경
## 2.1.1 컴퓨터의 주요 구성 요소
### 컴퓨터 하드웨어
```RAM```: 컴퓨터의 주기억 장치. 프로그램은 보조 기억장치에 저장되어 있는데 이를 실행하게 되면 프로그램의 복사본이 RAM에 적재되어 실행됨. RAM에서 실행하고 있는 프로그램이 종료될 때까지 RAM에 상주하게 되고, 프로그램이 종료되면 운영체제는 해당 프로그램을 RAM에서 삭제함. 휘발성. 임시저장소.
```CPU```: RAM으로부터 명령을 받아서 실행. 이러한 명령은 하드 디스크에 바이너리 코드로 저장되어 있음. X86-윈도우, ARM-MAC
```입/출력 장치```: 외부 장치로부터 컴퓨터와 상호작요용하게 함. 키보드, 마우스와 같은 입력장치로부터 정보를 RAM으로 읽어들이며, 모니터와 같은 출력장치로 정보를 내보내게 됨
```디스크```  : 파일, 디렉터리 또는 소프트웨어를 저장하는데 사용하는 장치. ex) HDD, SSD
### 운영체제 구성 요소
```커널(Kernel)```: 운영체제의 핵심
- 파일시스템 관리
- 하드웨어 장치 관리
- CPU 제어
- 메모리 및 프로세스 관리

```쉘(Shell)```: 유닉스/리눅스에서 사용자와 커널 사이의 인터페이스를 당당하는 프로그램. 사용자가 명령어를 내리면 이를 커널이 알아들을 수 있는 명령어로 해석해서 커널로 전달
```디렉터리(Directory)```: 파일을 가지고 있거나, 또 다른 디렉터리를 가지고 있는 그룹이며 이는 계층적으로 구성되어 있음. 디렉터리와 파일은 파일 시스템에 의해 관리됨

# 2.3 CLI 구문 (Command-Line Interface)
## 2.3.1 CLI 구문
  : 명령어를 내릴 때 옵션과 아규먼트를 사용하여 명령어의 동작을 달리할 수 있음
  ```명령어(command)```: 시스템에서 특정 작업을 하기 위해 실행하는 파일
  ```옵션(option)```: 명령어를 어떻게 실행할 것인지 지정, 일반적으로 대시(-) 문자 뒤에 옵션을 지정, 옵션은 대소문자 구분
  ```아규먼트(argument)```: 명령어 실행에 영향을 받는 파일, 디렉터리, 텍스트와 같은 항목
### CLI 제어 문자
  : CLI에서 명령어를 입력하는 도중 키보드의 특수 제어문자를 사용하여 화면의 출력을 중지시키거나 명령어를 삭제하는 등 제어를 할 수 있음
  ```Ctrl+C```: 현재 동작 중인 명령어 종료
  ```Ctrl+D```: 파일의 끝(End-Of-File) 또는 종료를 나타냄
  ```Ctrl+U```: 현재 명령어 라인을 모두 지움
  ```Ctrl+W```: 커서가 있는 곳의 단어를 지움
  ```Ctrl+S```: 화면 출력 중지
  ```Ctrl+Q```: 화면 출력 시작(Ctrl+S로 중지시킨 후)
# 2.4 매뉴얼 페이지 ★
  : 명령어의 사용법과 설명을 가지고 있는 매뉴얼 페이지(Man Page) 제공
  ```
  man command
  man option command
  man option filename
  ```
