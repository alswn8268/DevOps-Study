# 11.1 유닉스/리눅스 운영 환경의 아카이브
- `tar` 명령어: 여러 개의 파일로부터 아카이브 파일을 생성하거나 해제할 수 있으며, 테이프나 저장 장치 등의 미디어에 저장 가능
- `jar` 명령어: 여러 개의 파일을 하나의 아카이브 파일로 합칠 수 있음

# 11.2 tar 명령어를 이용한 아카이브
- 원래 tar 파일은 마그네틱 테이프 장치를 위한 것
```
tar function(s) archivefile filename(s)
```
|기능|설명|
|---|---|
|c|새로운 tar 파일을 생성|
|t|tar 파일의 내부 내용들의 리스트를 확인|
|x|tar 파일 해제|
|f|아카이브 파일이나 테이프 장치 지정|
|v|tar 명령어 수행 과정 자세히 출력|
|h|아카이브하려는 파일이 심볼릭 링크 파일인 경우 원본을 아카이브|


#### doc1.txt부터 doc5.txt까지 생성
```
[vagrant@user01 ~]$ touch doc{1..5}.txt
```

#### docs.tar 생성
```
[vagrant@user01 ~]$ tar cf docs.tar doc1.txt doc2.txt doc3.txt 
```

#### -v 옵션 추가로, 생성 과정을 표시
```
[vagrant@user01 ~]$ tar -cvf docs2.tar *.txt 
doc1.txt
doc2.txt
doc3.txt
doc4.txt
doc5.txt
file2.txt
```

#### tar 파일 내용물 확인 (f)
```
[vagrant@user01 ~]$ tar -tf docs.tar
doc1.txt
doc2.txt
doc3.txt
```

#### 아카이브 해제 (x)
-> 기본적으로 파일을 덮어쓰기 때문에 주의.
```
[vagrant@user01 ~]$ tar -xvf docs.tar
doc1.txt
doc2.txt
doc3.txt
```

#### 덮어쓰기 방지 옵션 (t)
```
[vagrant@user01 ~]$ tar -xkf docs2.tar
tar: doc1.txt: Cannot open: File exists
tar: doc2.txt: Cannot open: File exists
tar: doc3.txt: Cannot open: File exists
tar: doc4.txt: Cannot open: File exists
tar: doc5.txt: Cannot open: File exists
tar: file2.txt: Cannot open: File exists
tar: Exiting with failure status due to previous errors
```

#### 특정 디렉터리에 압축 해제 (C)
```
[vagrant@user01 ~]$ tar -xvf docs.tar -C /tmp/
doc1.txt
doc2.txt
doc3.txt
```
