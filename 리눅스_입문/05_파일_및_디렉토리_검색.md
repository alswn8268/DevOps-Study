# 5.1 파일 내용 검색
## grep 명령어
하나 이상의 파일에서 문자 패턴을 검색하고, 매칭되는 결과를 화면으로 출력함
> grep option pattern filename
- 파일 내용을 수정하지는 않음

|옵션|설명|
|---|---|
|-i|검색 패턴 대소문자 무시|
|-l|매칭되는 패턴이 있는 파일 이름 출력|
|-n|매칭되는 줄 번호 표시|
|-v|검색 패턴을 제외하고 검색|
|-c|검색 패턴과 매칭되는 줄 개수 출력|
|-w|단어 단위로 검색|

#### grep 명령어의 정규표현식 메타 문자
|메타 문자|용도|예제|결과|
|---|---|---|---|
|^|줄의 시작 지정|^solaris|solaris로 시작하는 줄|
|$|줄의 마지막 지정|solaris$|solaris로 끝나는 줄|
|.|한 문자 대치|s.....s|s로 시작하고, 5개의 아무문자, s로 끝남|
|*|아무 것도 없거나 여러 문자 대치|[a-z]*|소문자로 시작하는 모든 문자 또는 아무 것도 없음|
|[]|패턴 중 한 문자 대치|[Ss]olaris|Solaris 또는 solaris|
|[^]|패턴 중 제외할 한 문자|[^a-r]solaris|첫 문자가 a에서 r까지 제외하고 s에서 z까지 오는 문자|

## erep 명령어
기본적으로는 grep 명령어와 사용법이 동일하며, grep 명령어의 기능에 추가로 확장 정규 표현식을 지원
|메타 문자|용도|예제|결과|
|---|---|---|---|
|+|특정 문자 앞에 하나 이상의 문자 있음|[a-z]+tion|tion 문자 앞에 소문자의 문자 있음 (예: caution, mention 등)|
|x&#124;y|x 또는 y|root&#124;admin|root 또는 admin|
|(abc&#124;def)|abd 또는 def (문자열)|ha(ve&#124;ving)|have 또는 having|

### fgrep 명령어
fgrep 명령어는 grep 또는 egrep과 비슷하나, 정규 표현식이나 확장 정규 표현식이 무시됨


## 5.2 파일 및 디렉터리 검색
- 계층적인 디렉터리 구조에서 특정 파일 및 디렉터리를 검색하기 위해 find 명령어를 사용
- fing 명령어는 파일 이름, 크기, 소유자, 시간, 종류와 같은 조건을 사용하여 검색할 수 있음
> find pathname expression action

#### find 명령어의 인자
|형식|설명|
|---|---|
|경로(pathname)|검색을 시작할 절대경로 또는 상대경로|
|조건(expression)|검색 조건 지정|
|동작(action)|파일 검색 후 동작 지정|

#### find 명령어 조건(Expression)
|조건|설명|
|---|---|
|-name filename|특정 파일 이름 지정(메타문자 사용 가능)|
|-type|검색할 파일 종류 지정|
|-perm|특정 권한을 가진 파일 검색|
|-user LoginID|특정 사용자가 소유한 파일 검색|
|-size [+&#124;-]n|파일의 특정 크기 검색 (+n은 이상, -n은 이하, n은 정확한 크기)|
|-atime [+&#124;-]n|파일에 접근한 시간|
|-mtime filename|파일을 수정한 시간|

#### find 명령어 동작(Action)
|동작|설명|
|---|---|
|-print|터미널에 검색한 파일의 경로와 이름 표시 (기본 값)|
|-ls|터미널에 검색한 파일의 경로와 이름 뿐만 아니라 자세한 정보 표시|
|-exec command {} \;|검색한 파일에 특정 명령 수행|
|-ok command {} \;|-exec 결과와 비슷하나, 특정 명령을 대화형으로 진행|
