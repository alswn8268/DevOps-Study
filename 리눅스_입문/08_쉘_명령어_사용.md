# 8.1 쉘 메타문자 사용
## 8.1.1 경로 이름 메타 문자
### 틸드(~) 메타문자
  - 현재 로그인된 사용자의 홈 디렉터리
  - Bourne Shell 에서는 사용할 수 없음
> 틸드(~)와 사용자 이름: 지정한 사용자의 홈 디렉터리
> 틸드(~)와 더하기(+) 문자: 현재 작업 디렉터리
> 틸드(~)와 대시(-) 문자: 이전 작업 디렉터리

### 대시(-) 문자
- 이전 작업 디렉터리

## 8.1.2 파일 이름 대체 메타 문자
### 별표(*) 문자
  - 아무 것도 없거나 하나 이상의 문자를 대체하는 문자
  - 숨겨진 파일에는 해당되지 않음
### 물음표(?) 문자
  - 하나의 문자를 대체하는 문자
### 대괄호([]) 문자
  - 하나의 문자를 대체하는 문자의 범위 지정
  ex. [a-f]는 알파벳 a에서 f까지 문자의 범위를 의미
### 달러($) 문자
  - USER라는 이름 변수

## 8.1.3 인용부호 메타 문자
### 홑 따옴표 ('')
- 홑 따옴표 안에 있는 모든 메타 문자를 일반 문자로 취급
```
# echo '$USER'
$USER
```
### 이중 따옴표 ("")
- 달러($), 역 따옴표(`), 역슬래시(\) 세 개의 메타 문자를 제외한 나머지 메타 문자를 일반 문자로 취급

### 역 홑 따옴표(``)
- 특정 명령어의 결과 출력
```
echo `date`
Friday, January 10, 2014 09:39:26 AM UTC

```
## 8.1.4 방향 재지정 메타 문자
- 입력 장치 또는 출력 장치를 다른 곳으로 재지정
- 쉘의 표준 입력 장치: 키보드 / 표준 출력 장치: 모니터 / 표준 에러 장치: 모니터
- 사용되는 메타 문자: >, <, 파이프 문자(|)

### 표준 입력 재지정
```
cat < /etc/hosts # 표준 입력을 /etc/hosts로 재지정
```
### 표준 출력 재지정
```
ps > process_list # 표준 입력을 process_list로 재지정
ps >> process_list # 파일 내용 뒤에 출력 추가
```
### 표준 에러 재지정
```
command 2> /dev/null
```

### 파이프 문자(|)
- 명령어의 표준 출력을 또 다른 명령어의 표준 입력으로 받음
> command | command
```
ls -l /etc | wc -l
# ls -l /etc 명령어의 결과를 wc -l 명령어의 입력을 받아서 줄의 수(즉, 파일의 수)를 확인
```

# 8.3 사용자 초기화 파일
## /etc/profile
- root에 의해 관리되며, 시스템 전역에 걸쳐 환경을 설정하는 파일로, 모든 사용자는 시스템 부팅 시 적용됨
## ~/.profile
- 개별 사용자 홈 디렉터리의 .profile 파일은 해당 사용자가 로그인시 적용되는 초기화 파일. 이 파일에서는 Shell에서 사용할 Shell 변수나 터미널 설정 등을 할 수 있음
## ~/.bashrc 또는 ~/.bash_profile
- Bash Shell 실행 시 실행하는 초기화 파일

### Alias
```
[vagrant@user01 ~]$ alias lsd='ls -ld' # alias 설정

[vagrant@user01 ~]$ lsd testDir
drwxr-xr-x. 2 vagrant vagrant 6 Oct 14 02:27 testDir
[vagrant@user01 ~]$ ls -ld testDir
drwxr-xr-x. 2 vagrant vagrant 6 Oct 14 02:27 testDir
```
* 세션이 바뀌면 사라짐 -> bashrc에 추가
