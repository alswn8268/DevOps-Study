## AWS 제품 및 서비스 관련 용어와 개념

### 클라우드 컴퓨팅
- 온디맨드로 서비스 액세스
- 필요에 따라 컴퓨팅 리소스 프로비저닝
- 사용한 만큼만 요금 지불
#### 클라우드 컴퓨팅의 이점
- 종량제
- 거대한 규모의 경제로 얻게 되는 이점
- 용량 추정 불필요
- 속도 및 민첩성 향상
- 비용 절감 목표 실현
- 몇 분만에 전 세계에 배포

### AWS Global Infrastructure
- 리전, 가용 영역, 엣지 로케이션이 포함됨

### AWS 리전
- AWS가 데이터 센터를 호스팅하는 전세계의 실제 지리적 위치
- 리전은 3개 이상의 가용 영역(AZ)으로 구성됨
#### 리전 선택
- 지연 시간, 요금, 서비스 가용성, 데이터 규정 준수 요인을 고려해야 함

### 가용 영역(AZ)
- 가용 영역은 중복 전력, 네트워킹 및 연결 기능을 갖춘 하나 이상의 데이터 센터로 구성됨

### 엣지 로케이션
- 콘텐츠가 캐시되는 전 세계의 위치

## AWS 보안 및 AWS Identity and Access Management (IAM) 관련 주요 개념
- AWS 계정 및 리소스에 대한 보안 액세스를 관리할 수 있는 웹서비스

### IAM 기능
- 전 세계에서 사용가능
- AWS 서비스와 통합됨
- AWS 계정에 대한 공유 액세스
- Multi-Factor Authentication (MFA)
- 자격 증명 연동
- 무료로 사용

### AWS 루트 사용자
> AWS 계정 생성 (루트 사용자 ID 설정) -> 첫 번째 IAM 사용자를 생성한 후 다른 사용자 생성 권한 제공 
- 루트 사용자는 되도록 사용하지 않음

### IAM 사용자
- AWS 서비스 및 리소스와 상호 작용하는 사람이나 애플리케이션을 나타내는 자격증명

### IAM 정책
- AWS 서비스 및 리소스에 대한 권한을 부여하거나 거부하는 JSON 문서
- 보안 최소 권한의 원칙을 준수해야 함
- `Effect` / `Action` / `Resource`로 구성

### IAM 그룹
- 그룹에 할당된 정책을 상속하는 IAM 사용자 모음

### IAM 역할
- 다른 AWS 리소스나 서비스에 임시로 액세스하기 위해 특정 사용자나 서비스가 수임할 수 있는 자격 증명
