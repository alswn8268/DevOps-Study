# AWS에서 제공되는 데이터베이스의 이점
  - 목적별 데이터베이스
  - 규모에 따른 성능
  - 완전관리형
  - 우수한 보안성과 가용성
# 데이터베이스 유형
  - `관계형`: Aurora, RDS, Redshift
  - `키-값`: DynamoDB 
  - `인-메모리`: ElastiCache for Memcached, ElastiCache for Redis
  - `문서`: DocumentDB
  - `와이드 컬럼`: Keyspaces
  - `그래프`: Neptune
  - `시계열`: Timestream
  - `원장`: Quantum Ledger Database (QLDB)
## 관계형 데이터베이스
- 행과 열이 이뤄진 테이블 집합으로 구성된 데이터 레코드 모음

### 관계형 데이터베이스의 이점
- 복잡한 SQL 사용 가능
- 중복성 감소
- 널리 알려진 형식
- ACID 원칙 등

# Amazon Relational Database Service (Amazon RDS)
## Amazon RDS
> 고객이 AWS 클라우드에서 확장성이 뛰어나고 안전한 관계형 데이터베이스를 생성하고 관리할 수 있도록 지원하는 관리형 웹 서비스
- 일반적인 DB 관리 작업을 AWS에서 관리함
### DB 인스턴스
> DB 인스턴스는 클라우드에서 실행되는 격리된 DB 환경
- 단일 DB 인스턴스로 여러 DB를 보유하고 CLI나 관리 콘솔을 사용해 DB에 엑세스할 수 있음.
- RDS에서는 모든 표준 SQL 클라이언트 애플리케이션을 사용하여 DB에 엑세스할 수 있음. 하지만 호스트에 대한 직접 엑세스는 불가

### DB 인스턴스 클래스
- `표준`, `메모리 최적화`, `버스트 가능 성능`

### DB 인스턴스 스토리지
- `범용 SSD`, `프로비저닝된 IOPS`, `마그네틱`

### 데이터베이스 보안
> RDS DB 인스턴스는 VPC 내의 프라이빗 서브넷에 생성함. 웹 애플리케이션, 콘텐츠 관리 시스템 등에 의해 쿼리가 수행되는 경우 VPC의 격리된 경계 내에서 이를 수행하므로 퍼블릭 서브넷에서 DB 인스턴스가 인터넷에 노출되지 않음

### 데이터베이스 백업
- `자동 백업`: 기본, 최장 35일
- `스냅샷`: 35일 이상도 가능

### 데이터베이스 이중화
> RDS 다중 AZ를 사용하도록 설정하면 RDS는 이중화를 위해 다른 가용 영역에 DB의 보조 사본을 생성함.

# Amazon DynamoDB
## DynamoDB
> 완전 관리형 NoSQL 데이터베이스 서비스로서 원활한 확장성과 함께 빠르고 예측 가능한 성능을 제공하는 키-값 데이터베이스
- 서버리스
- 저장 중 암호화
- 데이터와 트래픽 자동 분산 -> 일관적이고 빠른 성능
- SSD에 저장, 리전의 여러 가용 영역으로 자동 복제 -> 고가용성, 데이터 내구성

### 핵심 구성 요소
- `테이블`, `항목`, `속성`
