# Amazon Elastic Compute Cloud (Amazon EC2)
## Amazon EC2
> 클라우드에서 안전하고 크기를 조정할 수 있는 컴퓨팅 용량을 제공하는 웹 서비스로, 이를 사용하면 Amazon EC2 인스턴스라고 하는 가상 서버를 프로비저닝할 수 있음
- 몇 분 내에 하나 이상의 EC2 인스턴스 프로비저닝 및 시작
- 워크로드 실행 종료 후 EC2 인스턴스 중지 또는 종료
- 시간이나 초 단위로 각 인스턴스 유형의 요금 지불 (최소 60초)
- 시작할 때 먼저 Amazon Machine Image(AMI)를 선택해야 함

### Amazon Machine Image (AMI)
- AMI에는 운영 체제, 스토리지 매핑, 아키텍처 유형, 시작 권한, 그리고 사전 설치된 추가 소프트웨어 애플리케이션이 포함됨
- 지정한 AMI는 루트 디바이스 볼륨에 복사됨
- 단일 AMI를 사용하면 동일한 구성으로 여러 EC2 인스턴스를 시작할 수 있음

### Amazon EC2 인스턴스 유형 예제
- `범용` / `컴퓨팅 최적화` / `메모리 최적화` / `가속 컴퓨팅` / `스토리지 최적화`

### Amazon EC2 요금
- `프리 티어` / `Savings Plans` / `전용 호스트` / `온디맨드 인스턴스` / `예약 인스턴스` / `스팟 인스턴스`

# Amazon Elastic Container Service (Amazon ECS), Amazon Elastic Kubernetes Service (Amazon EKS)
## 가상 머신과 컨테이너 비교
### 컨테이너
- 컨테이너는 애플리케이션 코드와 종속성을 패키징할 수 있는 표준 단위이며 어떤 운영 체제에서나 실행할 수 있음. 컨테이너 사용 시에는 모든 머신에서 호환성 문제 없이 애플리케이션을 동일하게 실행할 수 있으며 동작을 예측할 수 있고, 유지 관리 및 배포도 더욱 효율적임

### 컨테이너와 가상 머신과 다른 점
- 배포된 호스트와 같은 OS 및 터널 공유
- 몇 초만에 시작 가능

## AWS 컨테이너 오케스트레이션 서비스
### Amazon ECS
  - 컨테이너식 애플리케이션 실행 및 크기 조정
  - API 호출을 사용하여 Docker 지원 애플리케이션 제어
### Amazon EKS
  - Kubernetes 애플리케이션 실행 및 크기 조정
  - 패치, 노드 프로비저닝, 업데이트 자동화

# 서버리스
## 서버리스 
### 서버리스 컴퓨팅의 이점
- 서버를 프로비저닝 또는 관리할 필요가 없음
- 사용량에 따른 크기 조정
- 사용한 리소스만큼 비용만 지불
- 가용성 및 내결함성 기본 제공

## 서버리스 예시
- `AWS Fargate`: ECS, EKS 아키텍처 지원. 적절한 컴퓨팅 양 할당
- `AWS Lambda`: 서버를 프로비저닝하거나 관리하지 않고도 코드를 실행할 수 있는 컴퓨팅 서비스. 사용한 컴퓨팅 시간만큼만 비용을 지불함.
