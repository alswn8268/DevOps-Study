# Amazon Virtual Private Cloud (Amazon VPC)와 서브넷
## Amazon VPC
- VPC를 사용하면 데이터 센터의 기존 네트워크와 유사하게 AWS 클라우드에서 AWS 리소스를 위한 가상의 네트워크를 정의하고 프로비저닝할 수 있음
## 서브넷
- AWS 리소스를 저장하는 Amazon VPC 내의 더 작은 네트워크
- 서브넷이 인터넷에 연결되거나(퍼블릭), 연결되지 않도록(프라이빗) 라우팅을 구성할 수 있음.

### 인터넷 또는 원격 네트워크 액세스
- 인터넷 게이트웨이를 거쳐 인터넷을 통해 Amazon VPC에 연결 
- 프라이빗 서브넷의 AWS 리소스들은 NAT(Network Address Translation) 게이트웨이를 통해 인터넷에 연결하고 인터넷에서 해당 리소스에 연결되지 않게 할 수 있음
- 하나의 가용 영역에서 인터넷에 연결할 수 없게 되더라도 나머지 가용 영역의 리소스가 계속 인터넷에 연결될 수 있도록 각 가용 영역에 NAT 게이트웨이를 만들어야 함

> 온프레미스 데이터 센터와 Amazon VPC 간의 암호화된 가상 연결을 설정하려는 경우에는 가상 프라이빗 게이트웨이를 사용해야 함

### AWS Direct Connect
- 온프레미스 데이터 센터와 Amazon VPC 간에 안정적인 물리적 연결 설정

# 라우팅 테이블
## 라우팅 테이블
> VPC에 연결할 때는 라우팅 테이블의 정보에 따라 네트워크 트래픽이 전송되는 위치가 제어됨
- 네트워크 액세스 제어 목록, 보안 그룹, 방화벽 등 여러 계층으로 보안이 이루어짐
