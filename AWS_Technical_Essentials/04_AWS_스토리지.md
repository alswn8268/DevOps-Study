# 블록 스토리지
## 블록 스토리지의 유형
> 데이터가 크기가 고정된 블록으로 디바이스에 저장함
- `Amazon EC2 인스턴스 스토어`: EC2 인스턴스용 블록 수준 임시 스토리지. EC2 인스턴스를 호스팅하는 동일한 물리 서버에 사전 구성되고 미리 연결된 디스크 스토리지 블록으로 구성됨. EC2 인스턴스를 중지하거나 종료하면 스토리지 데이터가 손실됨 (임시)
- `Amazon Elastic Block Store (Amazon EBS)`: 가용 영역과 연결된 분리 가능 스토리지. EC2 인스턴스를 중지하거나 종료해도 데이터가 EBS 볼륨에 데이터를 계속 사용할 수 있음
증분 스냅샷 생성 가능

# 파일 스토리지
## Amazon EFS
> AWS 클라우드 서비스 및 온프레미스 리소스와 함께 사용되는 확장 가능한 파일 시스템
- 파일 스토리지 사용 시에 사용자, 애플리케이션, 서비스가 공유 파일 시스템의 데이터에 엑세스할 수 있음
- 여러 가용 영역에 걸쳐 수천 건이 넘는 EC2 인스턴스 연결 지원

## Amazon FSx
> 다양한 워크로드를 위해 클라우드에 매우 안정적이고 확장 가능한 파일 스토리지 솔루션을 제공하는 완전 관리형 파일 스토리지 서비스
- 완벽한 기능과 완벽한 관리
- 고가용성 및 데이터 보호
- 비용 효율성
- 하이브리드 지원

# 객체 스토리지
> 파일이 데이터, 메타데이터, 객체키 등으로 구성된 객체로 저장됨.
- 파일 업데이트시에는, 파일 조각 대신 전체 파일 객체가 업데이트 됨
## Amazon S3
> 객체 수준 스토리지 서비스. 인스턴스에 서비스를 연결하는 대신 웹을 통해 데이터를 검색
- 확장성이 뛰어나 데이터를 제한없이 저장 가능
- S3 내 객체외 최대 파일 크기는 5TB
- 객체는 컴퓨터의 디렉터리 또는 폴더와 비슷한 S3 버킷에 저장됨. 버킷은 리전 내에서 생성되지만 전역적으로 고유한 이름을 할당해야 함.
### Amazon S3 보안
- IAM 정책, 버킷 정책, S3 블록 퍼블릭 액세스, Amazon S3 암호화

### Amazon S3 스토리지 클래스
- Standard, Standard-IA, One Zone-IA, Glacier...

### Amazon S3 버전 관리
- 객체를 업로드할 때마다 객체의 새 버전 생성
- 버킷 수준에서 사용하도록 설정됨
- 기본적으로 사용 중지
