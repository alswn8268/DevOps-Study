## 생성형 AI 애플리케이션 수명 주기
> 사용 사례 정의 -> 파운데이션 모델 선택 -> 성능 개선 -> 결과 평가 -> 애플리케이션 배포

## 사용 사례 정의
- 해결해야 할 문제를 정의
- 관련 요구 사항을 수집
- 이해관계자의 기대치를 조정

### 생성형 AI 사용 사례의 주요 지표
- 생산성 향상
- 시간 절약
- 고객 만족도
- 품질 개선
- 비용 절감

## 파운데이션 모델 선택
> 최적의 성능을 달성하고 비즈니스 사용 사례의 특정 요구 사항을  충족하려면 사전 훈련된 올바른 모델을 선택해야 함

### 사전 훈련된 모델 선택 기준
- 비용, 제공 방식, 지연 시간, 다국어 지원, 모델 크기, etc...

### 평가 프로세스
> 가장 적합한 파운데이션 모델을 선택하려면 구조화된 평가 프로세스를 사용하는 것이 좋음

### 지속적인 최적화
> 파운데이션 모델을 선택하는 것은 지속적인 최적화를 수행하는 것이 중요

### 비용 절충
> 생성형 AI 애플리케이션을 개발할 때 조직은 비용과 성능의 균형을 신중하게 맞춰야 함

## 성능 개선
### 데이터 품질 및 사전 처리
  - 정확한 예측의 기반이 되는 정제되고 관련성이 높으며 잘 구조화된 데이터 세트기 있는지 확인
### 모델 선택 및 파인 튜닝
  - 하이퍼 파라미터 최적화와 함께 FM 성능을 극대화하는 핵심 단계
### 하이퍼 파라미터 최적화
  - 그리드 검색, 랜덤 검색 또는 베이지안 최적화와 같은 기법은 당면한 특정 문제에 대한 최적의 하이퍼 파라미터를 식별하는데 도움이 될 수 있음
### 효율적인 인프라 및 확장
  - 시간이 지남에 따라 FM 성능을 유지하고 개선하는 데 필수적
### 지속적인 모니터링 및 반복
  - 반복적인 개선을 통해 FM이 동적 환경에서 정확하고 관련성이 높은 상태를 유지할 수 있음

## 결과 평가
> 파운데이션 모델 성능을 평가하는 것은 모델이 비즈니스 목표를 효과적으로 충족하는지 확인하기 위한 AI 애플리케이션 수명 주기의 중요한 단계
### 모델 평가 지표
- 모델 정확도, 정밀도, 재현율, F1 점수, 지연 시간
### 테스트 데이터세트로 성능 평가
- 여러 테스트 데이터세트에서 모델이 어떻게 작동하는지 고려해야 함

## 애플리케이션 배포
> 개발에서 실제 사용으로의 전환 단계
### 주요 고려 사항
- 비용, 리전 선택, 서비스 할당량 등
