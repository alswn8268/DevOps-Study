# 프롬프트 엔지니어링 소개
## 프롬프트 엔지니어링
> 원하는 출력을 생성하기 위해 AI 모델에 사용하는 입력을 설계하고 개선하는 작업. 명확하고 구체적인 지침이 필요함
### 프롬프트 엔지니어링의 이점
  - 모델 기능 강화
  - 안전 조치 강화
  - 영역별 지식 제공
  - 고품질 출력 확보

# 프롬프트 설계 모범 사례
## 모범 사례
  - 명확하고 간결하게 작성
  - 관련 컨텍스트 포함
  - 특정 명령 사용
  - 결과 요구 사항 명시
  - 응답 예시 제공
## 프롬프트 구성 요소(components)
> 프롬프트를 구성하는 상위 범주로, 태스크의 전반적인 프레임워크 규정
- 태스크: AI 모델이 해결해야 하는 목표 또는 문제를 명확하게 정의
- 지침: 태스크에 접근하고 해당 태스크를 완료하는 방법을 자세히 안내
- 입력: 관련 통계, 배경 정보 또는 특정 시나리오와 같이 모델이 고려해야 하는  정보 또는 컨텍스트 지정
- 출력: 생성되는 콘텐츠의 원하는 구조와 표현을 간략하게 설명
- 어조 및 스타일: 어조 및 스타일 구성 요소 설정
- 제약 조건: 특정 주제를 피하거나 특정 지침을 준수하게 함

## 프롬프트 요소 (elements)
> 프롬프트 내의 특정 세부 정보 또는 하위 구성 요소
- 역할
- 주제
- 출력
- 토큰
- 구조
- 제외 사항
- 대상
- 문서
- 예
- 길이
- 형식
- 경계

## 프롬프트 엔지니어링의 구성 요소 및 요소
> 역할 정의, 태스크 지침, 단계별 설명, 서식
* 모델은 기본적으로 이전 대화 내용을 기억하고 있지 않음

# 고급 프롬프팅 전략
## 프롬프트 엔지니어링 기법
- `제로샷(Zero-shot) 및 퓨샷(Few-shot) 예제`
- `지식 기반(Knowledge Bases)`: 파운데이션 모델 + 지식 기초 (ex.회사 데이터)
- `사고의 사슬(CoT), 사고의 트리(ToT)`: 단계별 사고 접근 / 다양한 추론 경로를 탐색
- `검색 증강 생성 (RAG)`: 생성형 AI 모델 + 외부 지식 소스
- `도구 사용 프롬프팅 및 행동 지향적 프롬프팅`: AI에 외부 리소스나 API를 사용하게 하거나, 특정 작업 또는 프로세스를 실행하도록 지시함
- `제약 기법`: 부정적인 프롬프팅

# 모델 설정 및 파라미터
> AI 모델의 동작과 출력을 제어하는 데 중요한 역할을 함. 주요 파라미터는 온도, 상위 P, 상위 K 등이 있음
## 설정 개요
> 모델 설정 관리는 AI 성능을 최적화하는 데 매우 중요함
- 창의성 수준을 관리하고 관련 출력을 보장하려면 파라미터 설정해서 적절한 균형을 찾는 것이 중요함
## 생성형 AI 모델 파라미터
### 파라미터
> 텍스트 생성의 창의적인 측면에 영향을 미침
- `온도`: 모델 출력의 무작위성과 창의성을 제어. 낮을 수록 보수적이고 집중적이고 높을 수록 다양하고 예측이 힘듦
- `상위 P`: 누적 확률을 기반으로 단어 선택을 제한하여 다양성을 제어. 낮을 수록 확률 분포의 상위만 조금 고려하고, 높게 설정하면 상위를 많이 고려함
- `상위 K`: 가능성이 가장 높은 특정 단어 수에서만 단어 선택. 낮게 설정하면 모델이 가능성이 가장 높은 단어만 고려하도록 제한하고, 높게 설정하면 모델이 가능성이 가장 높은 단어들로 구성된 더 큰 풀을 고려
=> 온도(0.7), 상위 P (0.9) 및 상위 K (40~50)의 기본 값은 집중된 출력과 다양한 출력의 균형을 맞춘 자연스러운 값

### 추론 파라미터
> 생성된 콘텐츠의 전체 구조와 함계 관리
- `최대 길이`: 생성되는 토큰 수의 상한 설정
- `중지 시퀀스`: 모델에 생성을 종료하는 신호를 보냄
