# 2.1 확장 권한
## 확장 권한의 종류
- `setuid`: 파일을 실행한 사용자가 아니라 파일을 소유하고 있는 사용자의 권한으로 프로세스 실행.
  - 퍼미션의 s는 실행 권한이 설정되어있다는 뜻이고, S는 설정되어 있지 않다는 것을 의미
- `setgid`: 파일을 실행한 그룹 아니라 파일을 소유하고 있는 그룹 권한으로 프로세스 실행.
  - 주로 디렉터리에서 사용
- `sticky bit`: 디렉터리 내 다른 사용자 파일 삭제 제한. 디렉터리에만 설정할 수 있으며, 파일을 소유한 사용자만 삭제할 수 있음.

> which 명령: 실행 파일의 경로 출력

```
[test33@user01 ~]$ ls -l /usr/bin/passwd 
-rwsr-xr-x. 1 root root 32656 May 14  2022 /usr/bin/passwd

[vagrant@user01 ~]$ chmod 777 /tmp/testdir
[vagrant@user01 ~]$ ls -la /tmp/testdir # 퍼미션의 t는 sticky bit를 의미
total 4
drwxrwxrwx.  2 vagrant vagrant    6 Oct 16 04:46 .
drwxrwxrwt. 11 root    root    4096 Oct 16 04:46 ..
```

## 확장 권한 설정
- 심볼릭 방식으로 설정하는 것이 편리
- 심볼릭 방식의 가장 앞자리는 확장 권한을 의미

| 권한 설정 | 심볼릭 |
|---|---|
| setuid 설정 (해제) | u+s (u-s) |
| setgid 설정 (해제) | g+s (g-s) |
| sticky bit 설정 (해제) | o+t (o-t) |

```
[vagrant@user01 ~]$ chmod +t /tmp/testdir/ # sticky bit 설정
```
