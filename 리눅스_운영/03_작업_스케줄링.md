# 3.1 단일성 작업 예약
> 한 번만 실행되고 종료되는 작업
 
## at 명령
> at [option] time-spec
- at 명령을 사용하면 프롬프트가 `at>` 으로 전환됨
```
[vagrant@user01 ~]$ at
Garbled time
```
### 작업 추가
- 필드에 맞게 순서대로 작성하며, 구분자는 공백
> 분 시 윌 월 요일 명령어
```
[vagrant@user01 ~]$ at now +5min # 지금으로부터 5분 후에 실행
warning: commands will be executed using /bin/sh
at> echo "Hello World" 
at> echo "Hello World!!!" > ~/message.txt
at> <EOT> # Ctrl+D
job 2 at Thu Oct 16 05:16:00 2025

[vagrant@user01 ~]$ atq # 실행될 job 목록
2       Thu Oct 16 05:16:00 2025 a vagrant
```
### 작업 삭제
- 작업 삭제는 `atrm` 명령어 사용
```
[vagrant@user01 ~]$ atrm 2 # 2번 작업 삭제
```

### at 명령으로 등록한 작업 확인
```
atq
```

# 3.2 주기적인 작업 예약
## crond 데몬
- crontab 명령을 사용하여 예약한 주기적인 작업을 실행
-  vi /etc/crontab 도 사용 가능하나 /etc 경로의 아래에 있어서 함부로 수정하면 위험
> 분 시 일 월 요일 명령어
```
crontab -e
no crontab for vagrant - using an empty one
crontab: installing new crontab

# 매 분 현재 시간을 파일에 지속적으로 기록
* * * * * date >> ~/date_log.txt
# 매일 02:00 "안녕하세요~~"
0 2 * * * echo "안녕하세여"
```

## anacron
- 예약 작업이 실행되어야할 때 **시스템을 사용할 수 없으면** 해당 작업을 기억해 두었다가, 시스템이 정상적으로 사용 가능하거나 부팅이 완료됐을 때 일정 시간 이후 해당 작업 진행
- anacron과 cron은 별개의 매커니즘을 가졌기 때문에 혼용할 수 있는 개념이 아님
